**2026.12.9**

**New**

* Added friend activity stats to the **Dashboard Hero**, including how many of your friends are currently playing **VRChat** and how many are currently logged in on the **web**.
* Added a new **Recent Photos** dashboard item that shows the latest photos you have taken in **VRChat**.
* Added a new **Favorited Avatars** dashboard item that displays your favorite avatars in a shelf-style view.
* Added a new **My Avatars** dashboard item that displays your own avatars in a shelf-style view.
* Added a new **Friends Location (Small)** dashboard item that shows your friends' locations in a more compact layout instead of large cards.

**Changes**

* Added missing translations for several texts in **Settings** and across the **Dashboard**.
* Improved wording and localization consistency across multiple areas of the UI.
* Refined dashboard text presentation for better clarity and readability.
* Updated the **Friends Sidebar** to show the world names your friends are currently in. Previously, it only displayed labels such as **Private Instance** or **In World**.
* Moved **Current World** higher up in user profiles, placing it above **Badges**, **Representing**, and **Bio** to make profiles easier to navigate.
* Updated the username in the **"Welcome, X"** dashboard hero to use the accent color.
* Moved the **usr_xxx-xxx-xxx** ID badge next to the user badges in user profiles.
* Moved the **wrld_xxx-xxx-xxx** ID badge next to the other badges in world modals.

**Fixes**

* Fixed several localization issues.
* Fixed multiple small bugs related to **REST API** calls.
* Fixed minor issues affecting dashboard text and translation display.
* Fixed an issue where incorrect **Time Spent** values could be be added to friends after restarting **VRChat** while **VRCNext** was still open.
* Fixed an issue where incorrect **Time Spent** values could be added to worlds after restarting **VRChat** while **VRCNext** was still open.
* Fixed a bug that caused the **Activity Log** to automatically scroll to the bottom whenever a new event was logged. It will now remain at your current scroll position.
